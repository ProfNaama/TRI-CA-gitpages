Baseline Task Description Template

This default task page is intentionally simple. In a real study, researchers usually replace it with domain-specific instructions, stimuli, examples, or reference content shown to participants before or during the chat task.

Section A. What this page demonstrates

Each treatment group can point to a different HTML file, so this area is often used to vary task framing, instructions, examples, or supporting materials across experimental conditions without changing the application code.

Section B. Basic formatting examples

The HTML can include basic formatting such as bold emphasis, italics, underlined text, inline code snippets, links, lists, and simple highlighted notes.

You can also add links such as this example link and line breaks if needed.
A horizontal rule can help separate sections clearly.

Example note. This highlighted area can be used for instructions, reminders, or a short scenario that participants should keep in mind while interacting with the AI system.
Section C. What participants see next
  • The chat area appears directly below this task description.
  • Assistant messages show the selected agent name and avatar image.
  • User messages show the selected user name and avatar image.
  • Those names and avatar files are configured in treatment_groups_config.csv.
  • The participant can type messages, send them to the agent, and end the chat when finished.

Keep this structure if it is useful, or replace the file entirely when defining a real task page.
